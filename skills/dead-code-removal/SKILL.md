# 不要コード削除スキル

このスキルは、使用されなくなったコード・リソース・依存関係の特定と安全な削除方法を提供します。

## 有効化するタイミング

- コードベースのクリーンアップを要求された場合
- リファクタリング作業の一環として不要コードを削除する場合
- プロジェクトの保守性向上が必要な場合
- パフォーマンス改善のために未使用リソースを削除する場合
- セキュリティ向上のために不要な依存関係を削除する場合

## 未使用コード

- どこからも呼び出されない関数
- 定義されても参照されない変数
- インスタンス化されないクラス
- 実装されないインターフェース
- 参照されない型・構造体

## 到達不能コード

- 決して実行されない条件分岐
- return文以降のコード
- throw文以降のコード

## 冗長コード

- 同じ処理が複数箇所に存在
- 一度だけ使用される中間変数
- 処理内容が空の関数
- 実装されていない関数

## 非推奨コード

- deprecatedな関数・APIなどの廃止予定の機能
- サポート不要になったコード
- フラグで無効化されている機能

## 削除手順

小さな単位での削除を推奨する。

### 削除の優先順位

1. 未使用import/require
2. 未使用変数
3. 未使用private関数
4. 未使用クラス

## 安全な削除のチェックリスト

### コード削除前

- 静的解析ツールで正確に特定されているか
- 動的分析でも未使用が確認できているか
- リフレクション・動的呼び出しの可能性はないか
- 設定ファイルからの参照はないか
- ドキュメント・コメントで言及されていないか
